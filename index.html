  • Vamtlab 2.0  

    An app to aggregate and organize my old mock trial team's competition data.

    UVa's Mock Trial Team has been keeping performance data for as far back as I've been a member of the program. Aggregating it has been a pain, given that it's stored in one spreadsheet per tournament with inconsistent naming of people and places. I'd aggregated a bunch of the spreadsheets to try and run some scripts on them for cleaning and analysis, but at some point the data entry needs to get sent to a database. I'd built an initial version, but the data model was a trainwreck and it seemed easier to blow it away and start over than to fix it.

    • Route 53
    • Cloudfront
    • Angular 7
    • Angular Material
    • Okta
    • Django/Django Rest Framework
    • Postgres
    • Docker

  • Madabolic H Street  

    A static site for getting more information about my favorite gym.

    A friend asked me if I could throw together a quick static website ahead of the opening of a new Madabolic location in DC. I wasn't the original developer of the main website, but I figured it'd be best to stay on brand and use the output HTML from the main site. It looks like the main developers are using Laravel, and I don't think they'd ever hand over their source code to me, so the code here is semi-spaghetti without the original PHP. I was blown away by how easy it was to pop in a form submission to a google sheet and even more thrown by how developer friendly Facebook pixels are. If you haven't heard of MAD before, I'd strongly recommend trying it out.

    The Stack

    • S3
    • Route 53
    • JQuery
    • Google Sheets API
